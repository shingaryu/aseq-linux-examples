Linux hardware preparation
  • Check your hidraw devices:
    1 ls -al /dev/hidraw*

    You should see a list like this:

  • Plug your spectrometer into any USB port. Your system should detect it as a new /dev/hidraw[N] device:

  • Now you can check the device info, using the following command (replace [NUMBER])
    1 cat /sys/class/hidraw/hidraw[NUMBER]/device/uevent


  • By default only root can open, read and write to the hidraw ports. This should be changed.
    • Firstly, we should create a new group with r/w rights for this port (if you already have an additional group (besides root) with r/w access, just add your user to it, as shown in the next step)
      To allow working with our hidraw port, we can use the solution from https://unix.stackexchange.com/questions/85379/dev-hidraw-read-permissions
    • Create a .rules file (e.g. named “99-hidraw-permissions.rules”) within the /etc/udev/rules.d directory. This file should contain the following line:
      1 KERNEL=="hidraw*", SUBSYSTEM=="hidraw", MODE="0664", GROUP="plugdev"


    • Now you should add your current user to plugdev group. To do this, execute the following command:
      1 sudo gpasswd --add ${USER} plugdev

    • Reboot. Now your user should be in the plugdev group, and the hidraw ports should have r/w access for this group:
