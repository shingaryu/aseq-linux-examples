Build and install (currently Linux guide only)

CMake build tool

CMake is a multi-platform build tool that can generate build files for many different target platforms. See more info at http://www.cmake.org

Building the library and examples

The project settings used by CMake to generate the platform specific build files is called CMakeLists.txt.

You can work with it from the terminal as shown below, or open it with any IDE that supports Cmake - e.g. Qt Creator.

Building on Linux

  1. Install CMake 3.6 or greater: http://cmake.org/cmake/resources/software.html
  2. Install the dependencies
    Libudev headers and libraries are required to build libspectrometer library. To install libudev libraries on Ubuntu, and other Debian-based systems, run:
    1 $ sudo apt-get install libudev-dev
    On Redhat-based systems, run the following as root:
    1 yum install libudev-devel

  3. Run the deploy.sh script located in top source directory OR run its commands from the top source directory to generate the build files and build using the generated Makefile.
    If you do not want to generate the packages for library, development files (headers and static library) and [optionally] examples using the Cpack system, do not invoke the cpack command
    1 $ rm -rf build
    2 $ cmake -H. -Bbuild -DCMAKE_BUILD_TYPE=Release
    3 $ cd build
    4 $ make
    5 
    6 $ cpack
    This command creates a separate "build" directory thus keeping a clean source tree. CMake recommends to build "out-of-source" to keep the build files separated from the sources.
  4. If you prefer to install without using packages, use the following command or run checkinstall (highly recommended! ):
    1 $ sudo make install
    NOTE: In many cases sudo checkinstall is preferable to sudo make install as it keeps track of all files installed by a "make install" or equivalent, creates a custom Slackware, RPM, or Debian package with those files, and adds it to the installed packages database, allowing for easy package removal or distribution.

Available CMake options

  • WITHOUT_APPS
    This option allows to build libspectrometer library without examples. By default, main examples are built with the library (linked statically)
    To use this option, add it to the Cmake arguments:
    1 $ cmake -H. -Bbuild -DCMAKE_BUILD_TYPE=Release -DWITHOUT_APPS
  • WITH_EXTRA_SYSTEM_LINKED_APPS
    This option allows to build extra examples which are linked dynamically to the shared library libspectrometer after you install it (library is searched in /usr/local/lib and /usr/lib). By default, these examples are not built. Use it only after library installation, packaged or otherwise.
    1 $ cmake -H. -Bbuild -DCMAKE_BUILD_TYPE=Release -DWITH_EXTRA_SYSTEM_LINKED_APPS

  • CMAKE_INSTALL_PREFIX
    A common option you may want to give is to set the install path, same as –prefix= with autotools. It defaults to /usr/local.
    1 $ cmake -H. -Bbuild -DCMAKE_BUILD_TYPE=Release -DCMAKE_INSTALL_PREFIX:PATH=/usr
